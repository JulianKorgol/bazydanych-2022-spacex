Dokument jest charakteryzowany poziomem dostępu, to znaczy, że headadmin posiada dostęp do funkcjonalności admina,
bo posiada wyższe uprawnienia niż admin.

Hierarchia - najwyższy = najwięcej uprawnień:
- headadmin
- admin
- użytkownik



Uprawnienia:
	Tworzenie użytkowników => headAdmin 
	Wyświetlanie użytkowników => admin
	Tworzenie misji => admin 
	Dodawanie użytkowników do misji => admin 
	Wyświetlanie misji => użytkownik
	Wyświetlanie szczegółów misji => użytkownik
	Szczegóły użytkowników => admin ( + dany użytkownik - jeśli starczy czasu)
